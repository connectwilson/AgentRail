# Roadmap

`AgentRail` is moving toward a protocol surface that lets agents work with onchain systems using fewer assumptions, smaller payloads, and safer defaults.

## Near Term

- expand protocol registries beyond Aave BNB
- add more high-level methods for common DeFi tasks
- improve `receipt.decode` effect extraction
- add better protocol-native compact views
- improve method manifests and machine-readable schemas

## Mid Term

- broader multi-chain EVM coverage
- stronger pagination and projection controls
- better action planning for multi-step flows
- richer policy controls for writes
- better protocol-specific summaries and highlights

## Longer Term

- more protocol adapters
- better task-level execution flows
- stronger support for non-EVM chains through adapters
- a more formal protocol specification and versioning story

## Product Direction

The goal is not just to call contracts.

The goal is to make it easier for agents to:

- discover what exists
- understand what a call means
- execute safely
- recover from common failures
- explain what happened afterward
